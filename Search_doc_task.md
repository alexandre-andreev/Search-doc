Ниже план проекта: **семантический поиск по локальной библиотеке книг с интерфейсом через Codex**.

**Цель Проекта**
Сделать локальный поисковый движок для каталога книг, который умеет находить материалы **по смыслу**, а не только по точным словам. Пользователь общается с Codex обычным языком, Codex вызывает локальные команды проекта, получает результаты из индекса и возвращает понятный ответ.

Пример запроса:

```text
Найди книги про модульное тестирование, даже если там не написано unit testing
```

Результат:

```text
1. Искусство автономного тестирования с примерами на C#
2. The Art of Unit Testing
3. Python Testing with pytest
4. Learning Test-Driven Development
```

- Определить, какие части можно переиспользовать.


**Этап 2. Проектирование Семантического Индекса**
Что сделать:

- Выбрать формат хранения индекса.
- Рекомендуемый вариант: `SQLite`.
- Создать схему базы:
  - `books` - книги;
  - `chunks` - текстовые фрагменты;
  - `embeddings` - векторы;
  - возможно `index_runs` - история индексаций.
- Определить ключи для инкрементального обновления:
  - путь;
  - размер файла;
  - дата изменения;
  - SHA-256, если уже есть.
- Определить размер чанков:
  - 1500-3000 символов;
  - overlap 200-400 символов;
  - для PDF желательно хранить номер страницы.

Результат этапа:

```text
SQLite-схема и описание правил индексации.
```

**Этап 3. Извлечение Текста**
Что сделать:

- Расширить или переиспользовать текущий `utils/text_extractor.py`.
- Добавить режим полного или расширенного извлечения текста.
- Для PDF:
  - извлекать текст постранично;
  - сохранять номер страницы;
  - пропускать пустые страницы.
- Для EPUB/FB2/DOCX:
  - извлекать главы/абзацы;
  - приводить HTML/XML к чистому тексту.
- Для сканов:
  - пока помечать как "текст не извлечен";
  - OCR вынести в отдельный будущий этап.

Результат этапа:

```text

path, page/chapter, text.
```

**Этап 4. Нарезка На Чанки**
Что сделать:

- Создать модуль чанкинга.
- Делить текст на смысловые фрагменты.
- Не резать слишком мелко: иначе теряется контекст.
- Не резать слишком крупно: иначе embeddings становятся размытыми.
- Добавить overlap между соседними чанками.
- Сохранять:
  - `book_id`;
  - `chunk_index`;
  - `source_location`;
  - `text`;
  - `text_hash`.

Результат этапа:

```text
Для каждой книги создаются стабильные чанки, пригодные для поиска.
```

**Этап 5. Embeddings-Провайдер**
Что сделать:

- Создать единый интерфейс:

```python
embed_texts(texts: list[str]) -> list[list[float]]
embed_query(query: str) -> list[float]
```

- Поддержать минимум один backend:
  - OpenAI embeddings;
  - или локальная модель.
- Желательно предусмотреть несколько провайдеров:
  - `openai`;
  - `local_sentence_transformers`;
  - позже `gemini`/другие.
- Добавить батчинг, чтобы не отправлять по одному чанку.
- Добавить повторные попытки при ошибках API.
- Сохранять модель embeddings в БД, чтобы не смешивать разные векторы.

Результат этапа:

```text
Модуль, который превращает текстовые чанки и поисковые запросы в embeddings.
```

**Этап 6. Индексация Каталога**
Что сделать:

- Создать CLI-команду:

```powershell
python catalog.py semantic-index "C:\Users\alexa\YandexDisk\Книги"
```

- Команда должна:
  - просканировать каталог;
  - определить новые/измененные файлы;
  - извлечь текст;
  - нарезать на чанки;
  - посчитать embeddings;
  - сохранить всё в SQLite.
- Для уже проиндексированных книг ничего не пересчитывать.
- Показывать прогресс:
  - сколько файлов найдено;
  - сколько новых;
  - сколько обработано;
  - сколько пропущено;
  - сколько ошибок.

Результат этапа:

```text
Локальный индекс библиотеки:
cache/semantic_index.sqlite
```

**Этап 7. Семантический Поиск**
Что сделать:

- Создать CLI-команду:

```powershell
python catalog.py semantic-search "книги про модульное тестирование и mocks"
```

- Команда должна:
  - посчитать embedding запроса;
  - сравнить его с embeddings чанков;
  - найти top-N чанков;
  - сгруппировать по книгам;
  - посчитать общий score книги;
  - вернуть лучшие книги.
- В результат включить:
  - название/путь;
  - score;
  - лучшие фрагменты;
  - страницы/главы, если известны;
  - почему книга найдена.

Результат этапа:

```text
Быстрый поиск по смыслу через локальный индекс.
```
